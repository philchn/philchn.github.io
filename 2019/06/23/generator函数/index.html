generator函数

Posted on 2019-06-23 | In 前端
Symbols count in article: 3.6k | Reading time ≈ 3 mins.

概念

Generator函数是ES6提供的一种异步编程解决方案，语法上面看，Generator函数可以理解成一个状态机，封装了多个内部状态，执行一个Generator函数就会返回一个遍历器对象，可以依次遍历Generator函数内部的每一个状态。

特征

  1. function 关键字和函数之间有一个(*),内部使用yield表达式定义不同的内部状态。
  2. 调用Generator函数后，函数并不执行，返回的也不是函数运行结果，而是一个指向内部状态的指针对象。

使用

接下来是一个简单的例子

function* PhilGenerator (){
    yield "testA";
    yield "testB";
    yield "testC";
}
for(let value of PhilGenerator()){
    console.log(value);
}

这个例子的输出结果，如图所示

在例子中，并没有return语句，但结果中却得到了输出值。其实，调用generator函数并不会执行这个函数，而是会创建一个迭代器（interator）；

通过迭代器控制generator（生成器）

function* PhilGenerator(){
    yield "testA";
    yield "testB";
}

const philInterator = PhilGenerator();

const result1 = philInterator.next();
console.log(result1)
if(typeof result1 == "object" && result1.value == "testA" && !result1.done){
    console.log("testA log")
}

const result2 = philInterator.next();
console.log(result2)
if(typeof result2 == "object" && result2.value == "testB" && !result2.done){
    console.log("testB log")
}

const result3 = philInterator.next();
console.log(result3)
if(typeof result3 == "object" && result2.value == undefined && result2.done){
    console.log("No more log")
}

先看一下这个例子的输出结果

在调用生成器后，就会创建一个迭代器：const philInterator = PhilGenerator();迭代器用于控制生成器的执行。

迭代器暴露的基本接口是next方法，这个方法可以向生成器请求一个值，从而控制生成器：const result1 = philInterator.next();
next调用后，生成器开始执行，当执行到yield关键字时，会生成一个中间结果，然后返回一个新的对象，其中封装了结果值和一个指示完成的指示器。

每当生成一个值后，生成器会非阻塞的挂起，等待下一次的请求。当再次向生成器请求时const result2 = philInterator.next();挂起状态的生成器从上次离开的位置继续执行，直到再次遇到yield ，再次生成一个值，然后再次进入挂起状态…当第三次执行next，没有可执行的代码了，所以生成器返回一个value值为undefined，done属性为true，表明工作完成。

对迭代器进行迭代

通过调用暴露出来的next方法，我们能向生成器请求新值，next方法返回的既有value值，也有一个done属性，用于告知是否执行完成，我们利用这个done属性，可以通过whil循环来进行迭代

function* PhilGenerator(){
    yield "testA";
    yield "testB";
}

const philInterator = PhilGenerator();
let item;
while(!(item = philInterator.next()).done){
    if(item.value !== null){
        console.log(item.value);
    }
}

这就是第一个例子中for-of循环的原理

实际应用

下面是一个generator函数的一个用于生产ID的实际应用的例子

function* IdGenerator(){
    let id = 0;
    while(true){
        yield ++id;
    }
}

const idInterator = IdGenerator();

const getId1 = {id:idInterator.next().value};
const getId2 = {id:idInterator.next().value};
const getId3 = {id:idInterator.next().value};

通过这个例子可以得到不重复的id，需要生成一个新的id的时候调用一下idInterator即可，如果需要生成另外一组id序列，再重新初始化一个新迭代器即可。

与生成器交互

除了我们可以从generator函数中获取多个值，我们还可以向生成器中传值，从而实现双向通信。

向生成器发送值最简单的方法和其他函数一样，调用函数并传入实参

function* PhilGenerator(action){
    const msg = yield ("phil" + action);

    if(msg == "test"){
        yield (msg + action);
    }
}

const philInterator = PhilGenerator("ma");

const result1 = philInterator.next();
console.log(result1);

const result2 = philInterator.next("test");
console.log(result2)

打印结果为

在调用PhilGenerator时，传入了ma，第一次打印结果为{value: “philma”, done: false}应该很好理解，但第二次打印的却是{value: “testma”, done: false}，这是因为当test当做next的参数传入时，next的参数会用作整个yield表达式的值，即msg = yield (“phil” + action)中的变量msg最终值变为了test。

抛出异常

每个迭代器除了有next方法，还有一个throw方法。

function* PhilGenerator(){
    try{
        yield "phil";
        console.log("success log")
    }
    catch(e){
        console.log("error log",e)
    }
}

const PhilInterator = PhilGenerator();

const result1 = PhilInterator.next();
if(result1.value == "phil"){
    console.log("phil success log")
}
const result2 = PhilInterator.throw("fail msg");
